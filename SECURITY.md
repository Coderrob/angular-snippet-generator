# Security Policy

## Supported Versions

We take security seriously and actively maintain security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Angular Snippet Generator, please help us by reporting it responsibly.

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report security vulnerabilities by emailing:

- **Email**: [coderrob@outlook.com](mailto:coderrob@outlook.com)
- **Subject**: `[SECURITY] Vulnerability Report - Angular Snippet Generator`

### What to Include

When reporting a security vulnerability, please include:

1. **Description**: A clear description of the vulnerability
1. **Steps to Reproduce**: Detailed steps to reproduce the issue
1. **Impact**: Potential impact and severity of the vulnerability
1. **Environment**: Your environment details (OS, VS Code version, extension version)
1. **Proof of Concept**: If possible, include a proof of concept or exploit code
1. **Contact Information**: How we can reach you for follow-up questions

### Our Commitment

- We will acknowledge receipt of your report within 48 hours
- We will provide a more detailed response within 7 days indicating our next steps
- We will keep you informed about our progress throughout the process
- We will credit you (if desired) once the vulnerability is fixed and disclosed

### Security Best Practices

When using this extension, consider these security best practices:

- **Code Review**: Always review generated snippets before using them
- **Input Validation**: Validate any dynamic content in your Angular applications
- **Access Control**: Be mindful of what files the extension can access
- **Updates**: Keep the extension and VS Code updated to the latest versions

### Scope

This security policy applies to:

- The Angular Snippet Generator VS Code extension
- All source code in this repository
- Official releases and distributions

### Out of Scope

This policy does not apply to:

- Third-party dependencies (report to respective maintainers)
- VS Code itself (report to Microsoft)
- User-generated content or configurations

### Disclosure Process

1. **Report**: You report the vulnerability privately
1. **Confirmation**: We confirm receipt and begin investigation
1. **Fix**: We develop and test a fix
1. **Disclosure**: We coordinate public disclosure with you
1. **Credit**: We acknowledge your contribution (if desired)

### Recognition

We appreciate security researchers who help keep our users safe. With your permission, we will:

- Add you to our security acknowledgments
- Include you in release notes
- Provide a public thank you
